# 产品需求文档 (PRD) - nb-fishing

## 1. 产品概述
**nb-fishing** 是一款专为新不伦瑞克省 (New Brunswick, Canada) 垂钓爱好者设计的移动端地理位置法规查询工具。用户可以通过点击地图上的水域，即时获取该位置的垂钓种类、开捕日期、限制额度（Bag Limits）等官方规定。

### 1.1 背景
NB 省的垂钓法规因地理位置（水域）、鱼种和水域属性（是否为潮汐区）而异。官方 Web 应用功能强大但对移动端用户不够友好。本项目旨在通过响应式设计和优化的查询逻辑，提升现场查询的便捷性。

## 2. 目标用户
*   **本地渔民**：在垂钓现场快速确认当日法规。
*   **垂钓新手/游客**：不熟悉各水域分区，依赖地图定位获取准确信息。
*   **合规监管关注者**：确保垂钓活动符合最新法规，避免违规。

## 3. 核心功能需求

### 3.1 地图交互与定位
*   **地图渲染**：支持街道图与卫星图切换。
*   **点击查询**：用户点击地图任一点，系统应自动识别该点 50 米范围内的相关水域。
*   **GPS 定位**：一键跳转至用户当前位置并自动触发查询。

### 3.2 空间规则分析逻辑
*   **多图层并发查询**：系统需同时查询 5 组核心鱼种图层（Non-Sport, Trout, Atlantic Salmon, Landlocked Salmon, Smallmouth Bass）。
*   ** jurisdictional 智能过滤**：
    *   根据水域的 `IS_TIDAL` 属性自动过滤非竞技鱼类规则。
    *   `IS_TIDAL=1` (潮汐区) 匹配潮汐专属规则；`IS_TIDAL=0/2` (内陆/可能) 匹配内陆规则。
*   **季节有效性校验**：自动比对当前日期与法规中的 `OPEN_DATE` 和 `CLOSE_DATE`，给出 ✅ OPEN 或 🚫 CLOSED 状态。

### 3.3 结果展示 (UI State Machine)
*   **三段式抽屉 UI**：
    *   **PEEK (预览)**：显示水域名称及查询提示。
    *   **HALF (概览)**：展示各鱼种卡片及核心限制额度。
    *   **FULL (详情)**：滚动查看所有详细备注和法条原文。
*   **本地化支持**：自动将 API 的编码转换为易读的英文名称。

## 4. 技术性非功能需求
*   **结构灵活性 (API Self-Healing)**：系统应能自动探测官方 ArcGIS 图层 ID 的变动，通过图层名称动态匹配最新的 ID，确保在后台数据结构微调时仍能正常运行。
*   **数据一致性**：核心元数据（物种名、水域定义）需定期从 ArcGIS 服务器同步并缓存。
*   **响应性能**：空间查询应在 2 秒内完成。
*   **离线容错**：在网络不佳时，应保留上一次成功查询的缓存数据展示。

## 5. 成功标准
1. 用户在地图上点击任意主要水域，均能准确弹出该水域的官方法规。
2. 法规状态（开放/关闭）能随系统日期自动正确跳转。
3. 在移动设备上操作流畅，无明显卡顿。
